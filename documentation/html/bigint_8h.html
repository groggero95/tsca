◆ INT_SIZE

#define INT_SIZE

Nominal size (number of bit) of the bigint number. This value must be a power of two and the only supported values are:

  • 64
  • 128
  • 256
  • 512
  • 1024
  • 2048
  • 4096

Example:

#define INT_SIZE 128

◆ NUMB_SIZE

#define NUMB_SIZE   INT_SIZE/VAR_SIZE + 1

Length of the vector representing a bigint. The +1 indicates that we always add VAR_SIZE bits to the nominal size defined in INT_SIZE.
This is necessary to account for overflow and for some intermediate value which may require more bits before being reduced again to the nominal size.

◆ UMAX

#define UMAX

Maximum number representable by a limb variable, its value depends from VAR_SIZE. For Example if VAR_SIZE = 16 UMAX will be initialized to 0xFFFF

Remarks
  • This contsnt is not used anymore
  • This value is automatically defined for the supported size

◆ VAR_SIZE

#define VAR_SIZE

Size (number of bit) of the vector element componing a bigint This value must be within the following:

  • 8
  • 16
  • 32
  • 64

Example:

#define VAR_SIZE 64

◆ bigint_t

typedef struct bigint bigint_t

Structure to hold large fixed size unsigned integer.

This structure is used to represent unsigned integer with a high bit count. Due to the size (mutch bigger than 128 bit) there is not a predefined type able to represent and handle such integers.
For this reason we need to split them up in a vector, which is the only member of this structure
